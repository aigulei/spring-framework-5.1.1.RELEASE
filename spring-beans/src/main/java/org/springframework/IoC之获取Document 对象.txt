1：EntityResolver的作用就是，通过实现它，应用可以自定义如何寻找【验证文件】的逻辑
2：EntityResolver的作用是项目本身就可以提供一个如何寻找DTD声明的方法，即由程序来
实现寻找DTD声明的过程，如果我们将DTD文件放到项目的某处，在实现时直接将此文档读取
并返回SAX即可。
3：如果SAX应用程序需要实现自定义处理外部实体，则必须实现此接口，并使用#setEntityResolver(
EntityResolver entityResolver)方法，向SAX驱动器注册一个EntityResolver实例
4：public class MyResolver implements EntityResolver
5：xmlBeanDefinitionReader.setEntityResolver(new MyResolver())
